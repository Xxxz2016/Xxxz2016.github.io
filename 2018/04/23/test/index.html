建站相关

如果不以文字总结和记录，很多东西都无法深入理解，且容易遗忘，所以考虑开始写博客。

想搭个人博客，在网上搜了一圈，了解了两种主流的方式：Github Page 和 Wordpress

在搭建之前首先需要准备两个东西：

  1. 域名：这样才能通过域名访问，可以通过阿里云、腾讯云购买等，需要备案，也可以买不需要备案的域名
  2. 空间：用来存放网页
    • 云服务器、VPS等等区别是啥？

WordPress

WordPress is open source software you can use to create a beautiful website, blog, or app.

  • 大约30%的网站使用wordpress
  • 使用wordpress建站的步骤简介：
    1. 购买VPS
    2. 配置LAMP or LEMP
    3. 部署WordPress
      • 创建WordPress操作的用户和数据库
      • windows下解压到htdocs文件夹，Linux下需要配置好nginx和wp-config文件，浏览器访问安装页面：选择语言->设置标题与管理员用户名与密码以及电子邮件->安装完成
    4. 访问网站或进入后台管理界面
    5. 主题安装：
      • 将theme资源解压到wp-content/themes目录，并赋予权限
      • 进入后台管理界面更换主题（也可直接在管理界面下载安装）

Github Page

利用Github Page 搭建个人博客网站

  • Github为我们即提供了域名，又提供了空间，非常方便。
  • 使用Github Page搭建博客的步骤简介：
    1. 新建repository，需要按Github个人网站的规则命名：YOUR-GITHUB-USERNAME.github.io
    2. Settings -> Launch automatic page generator -> Continue to layouts
    3. publish page，即可通过域名访问
    4. index.html
    5. 域名

Hexo

Hexo中文文档

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  • Hexo是一个开源的静态博客生成器,用node.js开发,作者是台湾大学生tommy351。
  • 安装步骤简介：
    1. 安装 nodejs 和 git
    2. 使用 npm 命令安装 hexo
    3. 初始化指定文件夹
    4. 在 _config.yml 中配置网站信息

BootStrap

首先我觉得大多用bs的都不是全职前端开发，而是兼职前端的后端开发或设计师。

其次用不用bs其实看项目的性质。
如果说项目to b的，且规模不大，没有设计师，没太高性能要求，没有样式非要怎样才行，那用bs主要是为了省时间。后端工程师不用花精力学习浏览器兼容，细节样式等。

bs 只能说一个jquery 的样式库，还并不能说是框架。现在前端都是组件化开发，react、vue、angular都有对应自己风格的组件库，简单方便，契合自己所使用的框架。而且这些框架都是数据驱动业务的框架（mvvm），都不愿意去引入jquery去怼dom。就算是有类似于vue-strap等等利用本身框架属性写成的bs，也不会使用毕竟element-ui、iview、ant-design等等都非常棒
